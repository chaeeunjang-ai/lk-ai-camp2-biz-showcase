# Nova AI Camp2 Showcase

> 스크린샷, GIF, 스킬 파일 등 추가 파일이 있다면 같은 폴더에 함께 넣어주셔도 됩니다.

## ① 해결하고 싶은 문제

단계마다 이전 히스토리를 직접 찾아 복붙해야 했음. 상품 기획 → 랜딩 페이지 → 광고 소재 → 강의안 → CRM 문자까지 이어지는 5개 레이어가 각각 단절되어 있었고, 매 단계마다 "이전에 뭐라고 했더라"를 다시 찾는 반복 작업이 발생.

---

## ② 기존 방식의 병목

각 산출물이 파편화된 문서/메모/채팅에 흩어져 있어서 광고 카피를 쓸 때 기획안을, 강의안 쓸 때 광고 카피를, CRM 문자 쓸 때 강의안을 다시 찾아 붙여넣는 구조. 논리적 일관성이 사람 손에 달려 있었음.

---

## ③ 지향했던 방향성

데이터 → 기획 → 메시지 → 콘텐츠가 하나의 논리적 흐름으로 자동 연결되는 구조. 고객사 데이터만 넣으면 모든 하위 산출물이 같은 언어·논리·데이터를 기반으로 자동 생성.

---

## ④ 현재 구현 단계

전체 5레이어 중 1레이어 약 60~70%

| 레이어 | 상태 |
|--------|------|
| 00. 상품 기획 (기획안 에이전트) | 구조 완성, 원시데이터 입력 진행 중 |
| 01. 디자인 기획안 (랜딩/광고) | 미착수 |
| 02. 강의안 | 미착수 |
| 03. CRM | 페이션트퍼널에서 일부 진행 |

우두머리 기준: 원시데이터 B(설문) + A(수강생 프로필) 완성 → 타겟 분석·유료 상품 기획서 생성 직전 단계.

---

## ⑤ 추후 과제

- 우두머리 `02_상품_로드맵.md` 채우기 → 타겟분석 + 유료 상품 기획서 생성 테스트
- 01 디자인 기획안 레이어 자동화 설계 — 기획서가 완성되면 랜딩 페이지·광고 카피를 자동 파생시키는 구조
- 03 CRM 레이어 연결 — 기획 데이터가 사전/사후 문자 자동 생성까지 이어지는 파이프라인
- 루루이펙트·성장연구소 등 다른 고객사에 동일 구조 복제

---

## ⑥ 배운 점 / 소감

"산출물을 만드는 게 아니라 산출물들이 연결되는 구조를 만들면, 이후 작업이 빨라진다." 지금까지 한 작업의 핵심은 각 기획 파일이 서로를 참조하는 의존성 구조를 코드처럼 명문화한 것. 이 구조가 잡히면 01~03 레이어도 같은 방식으로 확장 가능함.
